第 2 步：安装 Winget 及其依赖（如果没有）
═══════════════════════════════════════════════════════════════════

在 PowerShell 7 中运行以下完整脚本：

try {
    $ProgressPreference = 'SilentlyContinue'
    $ErrorActionPreference = 'Stop'
    
    # 1. 检查并安装 VCLibs
    Write-Host "正在检查 VCLibs..." -ForegroundColor Cyan
    $vcLibsInstalled = Get-AppxPackage -Name "Microsoft.VCLibs.140.00.UWPDesktop"
    
    if (-not $vcLibsInstalled) {
        Write-Host "正在安装 VCLibs..." -ForegroundColor Cyan
        $vcLibsUrl = "https://aka.ms/Microsoft.VCLibs.x64.14.00.Desktop.appx"
        $vcLibsPath = Join-Path $env:TEMP "VCLibs.appx"
        Invoke-WebRequest -Uri $vcLibsUrl -OutFile $vcLibsPath -TimeoutSec 60
        Add-AppxPackage -Path $vcLibsPath
        Remove-Item $vcLibsPath -Force -ErrorAction SilentlyContinue
    } else {
        Write-Host "✅ VCLibs 已安装" -ForegroundColor Green
    }
    
    # 2. 检查并安装 WindowsAppRuntime
    Write-Host "正在检查 WindowsAppRuntime..." -ForegroundColor Cyan
    $runtimeInstalled = Get-AppxPackage -Name "Microsoft.WindowsAppRuntime*"
    
    if (-not $runtimeInstalled) {
        Write-Host "正在安装 WindowsAppRuntime..." -ForegroundColor Cyan
        $runtimeUrl = "https://aka.ms/windowsappsdk/1.8/latest/windowsappruntimeinstall-x64.exe"
        $runtimePath = Join-Path $env:TEMP "WindowsAppRuntime.exe"
        Invoke-WebRequest -Uri $runtimeUrl -OutFile $runtimePath -TimeoutSec 60
        $process = Start-Process -FilePath $runtimePath -ArgumentList "--quiet" -Wait -PassThru
        
        if ($process.ExitCode -ne 0 -and $process.ExitCode -ne 3010) {
            throw "WindowsAppRuntime 安装失败"
        }
        Remove-Item $runtimePath -Force -ErrorAction SilentlyContinue
    } else {
        Write-Host "✅ WindowsAppRuntime 已安装" -ForegroundColor Green
    }
    
    # 3. 检查并安装 Winget
    Write-Host "正在检查 Winget..." -ForegroundColor Cyan
    $wingetInstalled = Get-AppxPackage -Name "Microsoft.DesktopAppInstaller"
    
    if (-not $wingetInstalled) {
        Write-Host "正在安装 Winget..." -ForegroundColor Cyan
        $wingetUrl = "https://github.com/microsoft/winget-cli/releases/latest/download/Microsoft.DesktopAppInstaller_8wekyb3d8bbwe.msixbundle"
        $wingetPath = Join-Path $env:TEMP "Winget.msixbundle"
        Invoke-WebRequest -Uri $wingetUrl -OutFile $wingetPath -TimeoutSec 120
        Add-AppxPackage -Path $wingetPath
        Remove-Item $wingetPath -Force -ErrorAction SilentlyContinue
    } else {
        Write-Host "✅ Winget 已安装" -ForegroundColor Green
    }
    
    # 刷新环境变量
    $env:Path = [System.Environment]::GetEnvironmentVariable("Path","Machine") + ";" + [System.Environment]::GetEnvironmentVariable("Path","User")
    
    Write-Host "✅ Winget 配置完成！" -ForegroundColor Green
    
} catch {
    Write-Host "❌ 错误: $($_.Exception.Message)" -ForegroundColor Red
}

验证：
----------------------------
winget --version

⚠️ 如果命令未找到，关闭 PowerShell 重新打开

第 8 步：配置 PowerShell 远程管理（让 AI 完全控制）
═══════════════════════════════════════════════════════════════════

⚠️ 此步骤需要管理员权限

以管理员身份运行 PowerShell 7：

1. 启用 WinRM（Windows 远程管理）
----------------------------
Enable-PSRemoting -Force
Set-Item WSMan:\localhost\Client\TrustedHosts -Value "*" -Force
Restart-Service WinRM

2. 配置防火墙规则
----------------------------
New-NetFirewallRule -Name "WinRM-HTTP" -DisplayName "Windows Remote Management (HTTP-In)" -Enabled True -Direction Inbound -Protocol TCP -LocalPort 5985
New-NetFirewallRule -Name "WinRM-HTTPS" -DisplayName "Windows Remote Management (HTTPS-In)" -Enabled True -Direction Inbound -Protocol TCP -LocalPort 5986

3. 设置执行策略（允许运行脚本）
----------------------------
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope LocalMachine -Force

4. 配置 SSH Server（可选，更安全）
----------------------------
# 安装 OpenSSH Server
Add-WindowsCapability -Online -Name OpenSSH.Server~~~~0.0.1.0

# 启动并设置自动启动
Start-Service sshd
Set-Service -Name sshd -StartupType 'Automatic'

# 配置防火墙
New-NetFirewallRule -Name sshd -DisplayName 'OpenSSH Server (sshd)' -Enabled True -Direction Inbound -Protocol TCP -Action Allow -LocalPort 22

验证：
----------------------------
Test-WSMan localhost
# 应显示 WSMan 配置信息

ssh localhost
# 应能够连接（输入密码）

═══════════════════════════════════════════════════════════════════
第 9 步：配置 Windows Defender 排除项
═══════════════════════════════════════════════════════════════════

⚠️ 此步骤需要管理员权限

以管理员身份运行 PowerShell 7：

try {
    $ErrorActionPreference = 'Stop'
    
    Write-Host "正在配置 Windows Defender 排除项..." -ForegroundColor Cyan
    
    # 排除 Hermes 目录
    $hermesPath = Join-Path $env:USERPROFILE ".hermes"
    Add-MpPreference -ExclusionPath $hermesPath
    Write-Host "已添加排除路径: $hermesPath" -ForegroundColor Green
    
    # 排除 npm 全局目录
    $npmGlobalPath = Join-Path $env:USERPROFILE ".npm-global"
    Add-MpPreference -ExclusionPath $npmGlobalPath
    Write-Host "已添加排除路径: $npmGlobalPath" -ForegroundColor Green
    
    # 排除 Python 脚本目录
    $pythonScripts = Join-Path $env:USERPROFILE "AppData\Local\Programs\Python"
    Add-MpPreference -ExclusionPath $pythonScripts
    Write-Host "已添加排除路径: $pythonScripts" -ForegroundColor Green
    
    # 排除进程
    Add-MpPreference -ExclusionProcess "node.exe"
    Add-MpPreference -ExclusionProcess "python.exe"
    Add-MpPreference -ExclusionProcess "pwsh.exe"
    Write-Host "已添加排除进程: node.exe, python.exe, pwsh.exe" -ForegroundColor Green
    
    Write-Host "✅ Windows Defender 排除项配置完成！" -ForegroundColor Green
    
} catch {
    Write-Host "❌ 错误: $($_.Exception.Message)" -ForegroundColor Red
}

第 10 步：配置文件系统权限
═══════════════════════════════════════════════════════════════════

确保 AI 代理有完整的文件访问权限：

try {
    $ErrorActionPreference = 'Stop'
    
    Write-Host "正在配置文件系统权限..." -ForegroundColor Cyan
    
    # 获取当前用户
    $currentUser = [System.Security.Principal.WindowsIdentity]::GetCurrent().Name
    
    # 配置 Hermes 目录权限
    $hermesPath = Join-Path $env:USERPROFILE ".hermes"
    if (-not (Test-Path $hermesPath)) {
        New-Item -ItemType Directory -Path $hermesPath -Force | Out-Null
    }
    
    $acl = Get-Acl $hermesPath
    $accessRule = New-Object System.Security.AccessControl.FileSystemAccessRule(
        $currentUser,
        "FullControl",
        "ContainerInherit,ObjectInherit",
        "None",
        "Allow"
    )
    $acl.SetAccessRule($accessRule)
    Set-Acl -Path $hermesPath -AclObject $acl
    
    Write-Host "✅ 文件系统权限配置完成！" -ForegroundColor Green
    Write-Host "用户: $currentUser (完全控制)" -ForegroundColor Cyan
    
} catch {
    Write-Host "❌ 错误: $($_.Exception.Message)" -ForegroundColor Red
}

常见问题排查
═══════════════════════════════════════════════════════════════════

问题1: 命令未找到 (git/python/node/hermes)
----------------------------
原因：环境变量未刷新
解决：
1. 关闭当前 PowerShell 窗口
2. 重新打开 PowerShell 7 (运行 pwsh)
3. 如果仍然失败，重启计算机

问题2: Winget 安装失败
----------------------------
提示：如果看到 "已安装更高版本" 的警告，这是正常的，可以继续

问题3: WSL 安装失败
----------------------------
解决：
1. 确保 Windows 版本支持 WSL2
2. 启用虚拟化功能（BIOS 中）
3. 以管理员身份运行命令

问题4: Hermes Gateway 启动失败
----------------------------
排查步骤：
1. 检查日志: hermes gateway logs
2. 检查配置: hermes config list
3. 重新初始化: hermes init
4. 检查端口占用: netstat -ano | findstr :3000

问题5: 权限错误
----------------------------
解决：
1. 以管理员身份运行 PowerShell 7
2. 重新执行第 9、10 步
3. 检查用户账户控制 (UAC) 设置

问题6: Python pip 权限错误
----------------------------
解决：
1. 使用用户安装: pip install --user package_name
2. 或配置: pip config set global.user true
